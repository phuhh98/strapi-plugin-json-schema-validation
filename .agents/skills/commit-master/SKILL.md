# commit-master Skill

Automate atomic commit message generation for this workspace, always invoking the "caveman-commit" skill for concise, scoped, and convention-compliant commits. The skill checks the current task scope (from beads or user input), classifies changes by folder, and applies the caveman commit style for all commits.

---

## Workflow Steps

1. **Determine Task Scope**
   - Check for active beads issues/tasks.
   - If ambiguous, prompt user for current task or scope.

2. **Classify Changes**
   - Changes in `dev/`, `.beads`, `.specify`, `.agents` → `chore`
   - Changes in `plugin/` → feature, bugfix, or refactor (ask user if unclear)

3. **Compose Atomic Commits**
   - For each logical change, create a separate commit.
   - Always invoke "caveman-commit" skill for message generation.
   - Each commit message must include a concise subject line (≤50 chars) and a description below the title clarifying the changes. The description should be brief for small changes (1-2 sentences), but may be longer if the change is large or complex. There is no hard limit.
   - This skill is invoked automatically whenever the user requests to create a commit (e.g., "commit changes", "make a commit").

4. **Review & Confirm**
   - Present commit messages for user review.
   - Allow user to edit or approve.

5. **Finalize**
   - Commit changes with approved messages.

---

## Quality Criteria

- Each commit is atomic (one logical change per commit).
- Commit messages are generated by "caveman-commit" and match project conventions.
- Each commit message includes a subject line and a description clarifying the change (unless the subject is fully self-explanatory). The description can be more than 2 sentences if the change is large or complex.
- Chores and features are correctly classified by folder.
- User confirms or edits commit messages before finalizing.

---

## Decisions

- "caveman-commit" skill is always invoked for all commits.
- Chores: changes in `dev/`, `.beads`, `.specify`, `.agents`
- Features/bugfixes: changes in `plugin/`
- No bypass for caveman-commit unless user explicitly requests.
